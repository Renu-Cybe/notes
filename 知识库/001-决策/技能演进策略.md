# 技能演进策略

## 核心决策

**日期**: 2026-03-31

**决策**: 不采用版本管理，采用"实装后独立演化"策略。

## 策略说明

```
外部技能 ──复制──→ 项目技能 ──skill-creator──→ 优化版本
    │                    │
    └─ 不再同步 ──────────┘
```

### 规则

1. **一次复制，永久独立** — 外部技能进入项目后，按项目需求独立演化
2. **skill-creator 是演进引擎** — 负责优化、重构、增强
3. **无需担心源目录更新** — 项目技能自成体系

## 已实装技能清单

| 技能 | 源目录 | 实装日期 | 状态 |
|------|--------|---------|------|
| **wewrite** | `F:\项目分析\wewrite` | 2026-03-31 | ✅ 已实装，独立演化 |

## 后续演进

- 使用 **skill-creator** 技能持续改进项目内技能
- 如需新功能，在项目中直接修改，不回同步到源目录
- 源目录可作为参考，但不做为依赖

## 决策背景

CTO-技术决策讨论结果：不需要技能版本管理，skill-creator 可以优化改进我们的技能，外面的技能实装进来就成了我们的技能。
