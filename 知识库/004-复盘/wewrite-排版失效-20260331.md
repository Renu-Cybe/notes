# wewrite 排版失效复盘

> 日期：2026-03-31
> 问题：文章发布后排版无效
> 知识库同步：[[skills/wewrite/references/lessons-learned]]

---

## 现象

使用 wewrite 发布公众号文章后，排版无效——文章没有应用任何主题样式，呈现为纯文本格式。

## 根因

从源目录 `F:\项目分析\wewrite` 复制到工作目录 `F:\Obsidian\skills\wewrite` 时，关键文件丢失：

| 文件/目录 | 应有状态 | 实际状态 |
|-----------|----------|----------|
| `toolkit/themes/*.yaml` | 16个主题定义文件 | 空目录 |
| `toolkit/converter.py` | 548行完整版（WeChatConverter 类） | 108行残缺版（简单文本替换） |
| `toolkit/theme.py` | 197行完整版（Theme 类 + CSS 解析） | 仅文件头 |

## 修复

从源目录完整复制：
1. `toolkit/themes/*.yaml` - 16个主题文件
2. `toolkit/converter.py` - WeChatConverter 完整实现
3. `toolkit/theme.py` - Theme 类 + CSS 解析

修复前后对比：

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| HTML 大小 | ~5000 字符 | ~16000 字符 |
| CSS 样式 | 无 | professional-clean 主题完整样式 |
| 内联样式 | 无 | 包含 `style="..."` |
| Dark Mode | 无 | 包含 `data-darkmode-*` |
| 代码块高亮 | 无 | 有 |

## 预防措施

### 复制后验证清单
```bash
# 检查文件数量
ls toolkit/themes/*.yaml | wc -l  # 应该 = 16

# 检查文件行数
wc -l toolkit/converter.py        # 应该 ~548
wc -l toolkit/theme.py            # 应该 ~197
```

### 发布前检查
- [ ] 输出 HTML 是否包含 `style="..."` 内联样式
- [ ] 是否包含 `data-darkmode-*` 属性
- [ ] 代码块是否有语法高亮

### 配置备份
- [ ] API Key 已正确写入 `config.yaml`（image.api_key）
- [ ] 微信配置（appid/secret）已验证
- [ ] 主题配置（theme）已指定
