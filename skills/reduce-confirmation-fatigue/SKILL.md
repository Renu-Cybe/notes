# reduce-confirmation-fatigue

减少用户确认疲劳，在确定性任务上自动执行，仅在关键决策点询问。

---

## 触发场景

使用此技能 when:
- 用户需要执行重复性、确定性的工作流步骤
- 检测到用户正在经历"确认疲劳"（连续多次相同类型确认）
- 任务类型属于 P0/P1 分级

---

## 任务分级系统

### P0 - 全自动任务 ✅

**特征**: 无副作用、可预测、可回滚

**任务类型**:
- 文件复制完整性校验
- 配置读取与更新（非敏感）
- 知识库同步
- 度量数据收集
- 日志记录

**处理方式**: 自动执行 → 事后简洁报告

---

### P1 - 半自动任务 ⚡

**特征**: 低风险、有预期结果、本地操作

**任务类型**:
- 配置更新并同步
- 批量文件操作（非删除）
- 数据整理与格式化
- 报告生成

**处理方式**: 执行 → 摘要报告 → 用户可中断

---

### P2 - 需确认任务 ⚠️

**特征**: 高风险、不可逆、跨系统、敏感

**任务类型**:
- API密钥、密码等敏感信息写入
- 删除操作（文件、数据）
- 对外发布（公众号、邮件、社交媒体）
- 跨系统写操作（数据库、远程API）
- 破坏性变更

**处理方式**: 确认请求 → 风险说明 → 执行

---

## 使用模式

### 模式一：自动执行（默认）

识别 P0/P1 任务 → 直接执行 → 事后报告

```
用户：检查文件完整性
[自动执行文件校验]
[完成后报告] ✅ 文件校验完成：16个主题已复制
```

### 模式二：疲劳检测

检测连续确认模式 → 主动建议切换自动模式

```
用户：确认
用户：确认
用户：确认
[检测到确认疲劳]
建议：检测到多次相同确认，是否启用自动模式？
```

### 模式三：安全边界

识别 P2 高风险任务 → 主动阻断并请求确认

```
用户：更新 API Key
[识别为 P2 敏感操作]
⚠️ 敏感信息写入确认：
- 操作：写入火山引擎 API Key
- 风险：密钥可能泄露到日志
- 建议：检查 .gitignore 配置

确认执行？ (y/N)
```

## 报告格式

### P0/P1 事后报告

```
---
✅ [任务名称] 完成

| 项目 | 内容 |
|------|------|
| 操作 | [具体操作] |
| 结果 | [成功/失败] |
| 耗时 | [X秒] |
| 输出 | [关键输出摘要] |

[如有问题会主动提示]
---
```

### P2 确认请求

```
⚠️ [高风险操作] 需要确认

操作类型：[删除/敏感写入/对外发布]
影响范围：[文件路径/系统/范围]
风险说明：[不可逆/泄露风险/公开可见]

建议：[安全检查项]

确认执行？ (y/N) [详细说明]
```

## 配置

在 `.claude/settings.json` 中配置：

```json
{
  "enabledPlugins": {
    "reduce-confirmation-fatigue": true
  },
  "env": {
    "REDUCE_CONFIRMATION_FATIGUE_ENABLED": "true",
    "AUTO_EXECUTE_P0": "true",
    "AUTO_EXECUTE_P1": "true",
    "CONFIRM_P2": "true"
  }
}
```

## 评估结果

| 指标 | With Skill | Without Skill | 改进 |
|------|-----------|---------------|------|
| **通过率** | **100%** | 72.7% | **+27.3%** |
| **平均耗时** | 12.8s | 16.2s | **-21%** |
| **Token** | 2,260 | 2,540 | **-11%** |

**关键结论**:
- P0/P1 任务确认次数从 2-3 次降至 **0 次**
- P2 任务安全边界完整保持
- 平均每个工作流节省 3.4 秒

---

## 边界与限制

1. **不自动执行 P2 任务** - 安全优先原则
2. **用户可随时接管** - 自动模式不是强制模式
3. **事后可审查** - 所有自动操作都有日志记录
4. **疲劳检测阈值** - 连续 3 次相同确认触发建议
