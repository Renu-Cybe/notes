为什么大多数人卡在第一步

我见过太多人了，买了各种 AI 课，收藏了一堆教程，转头就忘。

核心问题不是"AI 能做什么"，而是"我该让 AI 帮我做什么"。

就像给你一把很锋利的刀，你不知道要切什么，那刀再好也是摆设。

讲真，大多数人用 AI 的方式从一开始就错了——他们把 AI 当成一个"问答机器"，问一句答一句。

但 AI 真正厉害的地方是：它能陪你把一个模糊的想法，慢慢磨成一个具体的东西。

比如——写一个小工具。

Claude Code 是什么

好，先说说今天要用的工具。

Claude Code 是 Anthropic 出的一个命令行工具。说白了，就是让你在终端里直接跟 Claude 聊天，而且这个 Claude 能直接操作你电脑上的文件。

你可以把它理解成一个"住在终端里的 AI 助手"。

跟在网页上聊天不一样的是，它能：
- 直接读取你项目里的代码
- 帮你改文件
- 执行命令
- 跑测试

就像你旁边坐了个工程师，你指哪它改哪。

这玩意儿对新手来说可能听起来有点硬核。但别急， 30 分钟，我带你跑一遍。

安装其实没那么难

打开你的终端。

Windows 用户按 Win + R，输入 cmd，回车。

Mac 用户按 Cmd + 空格，输入 Terminal，回车。

然后输入这行命令：

npm install -g @anthropic-ai/claude-code

嗯，就一行。

如果你电脑上没有 npm ，先装个 Node.js ，去官网下载安装包，下一步下一步就行。

装完之后，输入：

claude

第一次启动会让你登录，按提示走就好。

就这么简单。我见过安装过程最友好的 CLI 工具之一。

写你的第一个小助手

好，现在我们来做点有意思的。

假设你经常需要把一些长文章总结成要点，但又懒得自己看。我们来让 AI 帮你做这件事。

在终端里，直接跟 Claude 说：

帮我写一个 Python 脚本，输入一篇文章，输出 5 个要点摘要

Claude 会问你几个问题，比如要处理什么格式的文章、输出要多长之类的。

回答完之后，它会直接帮你创建一个 summarize.py 文件。

然后你可以让它测试一下：

用这篇文章测试一下，帮我看看效果

它会自己跑脚本，看结果，发现问题，改代码。

整个过程你就像在跟一个人聊天。

"这里不太对，摘要太长了。"

"行，我改短一点。"

"能不能加个关键词提取？"

"好，加上。"

就像打游戏做任务一样，一步一步，你的小助手就成型了。

我踩过的坑

说实话，一开始我也不太会用。

第一次用的时候，我跟它说"帮我写个爬虫"，它直接给我生成了一堆代码，我复制粘贴，跑起来就报错。

然后我又问它为什么报错，它又给了一段修复代码，还是不行。

来来回回搞了半小时，我才反应过来——我压根没说清楚我要爬什么网站、要什么数据。

后来我学会了，跟 AI 沟通就像跟人沟通一样：

先把需求说清楚，再让它动手。

它会问你问题，你认真回答。它会提建议，你判断要不要采纳。

这不是"我问你答"，是"我们俩一起把事情做出来"。

接下来你能做什么

30 分钟，你已经能跑通一个简单的工具了。

接下来，你可以让 Claude Code 帮你：

•把刚才的脚本加个图形界面
•打包成可以直接双击运行的程序
•加上更多功能，比如支持 PDF 、网页链接

或者，你可以想想自己平时有哪些重复性工作，试着让 AI 帮你自动化。

每做成一个小工具，你对 AI 的理解就深一层。

慢慢地，你会发现——AI 不是用来"聊天"的，是用来"干活"的。

而你会从"知道 AI 有用"变成"真的在用 AI 帮自己省时间"。

这就叫 AI 平权。

不是技术大牛的专利，是你的日常工具。
