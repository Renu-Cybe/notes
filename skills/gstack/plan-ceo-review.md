# Plan CEO Review

Rethink the problem. Find the 10-star product hiding inside the request.

## Four Modes

| Mode | When to Use |
|------|-------------|
| **Expansion** | User described a narrow solution to a big problem. Expand to the full opportunity. |
| **Selective Expansion** | Expand some parts, hold others. Be explicit about what expands and what doesn't. |
| **Hold Scope** | The user's scope is right. Focus on execution quality, not expansion. |
| **Reduction** | User's scope is too big. Reduce to the narrowest viable wedge. |

## The 10-Star Framework

Don't settle for "good enough." What does a 10-star experience look like?

**Example (Airbnb):**
- 1 star: Find a place to stay
- 5 stars: Better than a hotel
- 10 stars: Magical experiences you can't get anywhere else

## 10-Section Review

1. **Problem Importance** (1-10)
2. **Solution Elegance** (1-10)
3. **Market Size** (1-10)
4. **Defensibility** (1-10)
5. **Execution Risk** (1-10, higher is better = lower risk)
6. **Team Fit** (1-10)
7. **Distribution Path** (1-10)
8. **Monetization Clarity** (1-10)
9. **10-Star Potential** (1-10)
10. **Overall Conviction** (1-10)

For each section:
- Score it
- Explain what a 10 looks like
- Gap analysis: why isn't this a 10?

## Execution Flow

### Phase 1: Read Design Doc

Read the design doc from `/office-hours` or user input.

### Phase 2: Determine Mode

Assess which mode applies:
- Did user describe a symptom or the underlying problem?
- Is their scope too big, too small, or just right?

### Phase 3: Run 10-Section Review

Score each section 1-10. Be honest — most ideas score 4-6.

### Phase 4: Identify Gaps

For low scores, identify what's missing to get to 10:
- Problem Importance 5 → "Find 3 users who've experienced this pain in the last week"
- 10-Star Potential 4 → "Consider what would make this magical vs merely convenient"

### Phase 5: Write Recommendations

Update the design doc with:
- Mode decision (Expansion/Selective/Hold/Reduction)
- 10-section scores and analysis
- Specific recommendations to improve weak scores
- Revised scope if applicable

## Output

- Mode decision with rationale
- 10-section scores
- Gap analysis for low scores
- Updated design doc
- Go/No-go recommendation

## Trigger Phrases

- "Review my plan"
- "Is this a good idea?"
