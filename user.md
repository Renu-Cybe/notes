# User Profile

## Preferences
- Boundaries: 权限隔离原则：禁止越权修改系统与用户数据、人类确认原则：关键操作必须显式授权、身份锚定原则：禁止伪装或隐藏意图
- Goals: 创造可以用agent帮助我的复利工作流
- Output format: 报告
- Memory allowed: 项目上下文与配置、决策、偏好与约定、学习与知识积累、任务与演进历史
- Memory forbidden: 1. 个人身份与敏感隐私信息 2. 不稳定的、临时的或过时的上下文 3. 与项目核心演进无关的外部信息 4. 可能引发循环或冲突的"元指令"
- Project entry: 阅读项目概要与核心文档 配置本地开发环境 探索代码结构与项目约定
- Organization: 主干：阶段目录 分支：主题目录 原料库：日志
- Default capture: 第一步：默认存放地 — "收件箱" 第二步：定期处理与精炼 第三步：最终位置结构化的知识库
- Archive policy: 第一步：总结与记录（最关键步骤） 第二步：更新项目状态文件 第三步：知识整合（进阶）第四步：原始任务项的处置
