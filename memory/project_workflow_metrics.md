# 工作流量化度量体系

**决策日期**: 2026-03-31
**决策类型**: CTO-技术决策

## 核心原则

> 不能度量就无法优化，不能优化就无法复利。

## 四维指标体系

### 1. 效率指标（时间维度）

| 指标 | 定义 | 计算方式 | 目标值 |
|------|------|---------|--------|
| **任务耗时** | 完成单次工作流的时间 | 人工耗时 - Agent耗时 | 降低80% |
| **并行度** | 同时执行的工作流数 | 并发技能数 | ≥3个 |
| **响应时间** | 需求→首版产出的时间 | 指令发出→首版交付 | <30分钟 |
| **吞吐量** | 单位时间完成工作量 | 任务数/天 | 持续提升 |

### 2. 质量指标（输出维度）

| 指标 | 定义 | 衡量标准 | 示例（wewrite） |
|------|------|---------|----------------|
| **一次通过率** | 无需修改直接交付的比例 | 发布无需人工修改次数/总次数 | ≥70% |
| **修改轮次** | 初稿到终稿的迭代次数 | 平均修改次数 | <2轮 |
| **合规率** | 符合规范要求的占比 | 格式/内容检查通过数/总检查项 | 100% |
| **去AI痕迹度** | 人工感检测得分 | 12维度验证通过 | 全通过 |

### 3. 复利指标（积累维度）

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|---------|------|
| **知识库增长率** | 记忆/决策条目新增速度 | 月新增条目数 | +10条/月 |
| **技能复用率** | 同一技能被调用频次 | 调用次数/周 | 持续提升 |
| **配置沉淀率** | 临时→永久配置转化 | 永久配置数/总配置数 | ≥80% |
| **自动化覆盖率** | 无需人工介入的步骤占比 | 自动步骤/总步骤 | ≥90% |

### 4. 业务指标（结果维度）

以公众号工作流为例：

```
输入指标 → 过程指标 → 结果指标

选题质量   →  成文速度   →  发布频率
  ↓            ↓             ↓
素材真实性  排版合规率     阅读量
  ↓            ↓             ↓
热点匹配度  去AI痕迹度    关注转化率
```

| 阶段 | 指标 | 目标 |
|------|------|------|
| **输入** | 热点匹配度 | ≥80% |
| **输入** | 素材真实性 | 100%（禁止编造）|
| **过程** | 成文速度 | <30分钟 |
| **过程** | 排版合规率 | 100% |
| **结果** | 周发布量 | ≥2篇 |
| **结果** | 关注转化率 | >0%（从0开始）|

## 度量频率

| 频率 | 内容 | 用途 |
|------|------|------|
| **每次执行** | 耗时、通过率、错误类型 | 实时监控 |
| **每周** | 技能复用率、吞吐量 | 趋势分析 |
| **每月** | ROI计算、知识库增长 | 战略复盘 |

## ROI 计算模型

```
工作流价值 = (节省人工时间 × 时薪) - (维护成本 + 配置成本)

复利价值 = 知识库资产价值 + 技能复用节省的总时间
```

## 实施检查清单

- [ ] 为每个工作流建立度量 baseline
- [ ] 配置自动化数据收集（如执行日志）
- [ ] 设定每个指标的目标值和阈值
- [ ] 建立每周/每月 review 机制
- [ ] 将度量结果反馈到技能优化（skill-creator）

## Why

铁三角中的"复利"需要可验证的数据支撑。没有度量，就无法判断工作流是否真正创造了复利价值。

## How to apply

1. **新建工作流时** — 同时定义度量指标
2. **执行工作流时** — 自动/手动记录数据
3. **复盘时** — 用数据驱动优化决策
4. **优化后** — 对比度量验证改进效果
