# 主动记录关键信息

**规则**：当用户提供 API Key、密钥、关键配置参数时，立即主动记录到相应配置文件，并明确告知用户已记录。

**Why**：
- 2026-03-31 用户提供了火山引擎 API Key 用于 wewrite 封面图生成
- 我使用了该 key 但没有主动确认是否已正确记录到 config.yaml
- 后续用户询问时才检查发现 config.yaml 中的 image.api_key 是错误的值
- 这增加了用户的认知负担，也造成了潜在的错误风险

**How to apply**：
1. 收到 API Key/密钥/关键配置 → 立即写入对应配置文件
2. 写入后明确告知："已记录到 config.yaml：image.api_key = ***"
3. 使用前验证配置完整性，如有问题主动修复并报告
4. 关键配置变更同步记录到知识库复盘目录

**触发场景**：
- 用户提供微信 appid/secret
- 用户提供图片生成 API Key
- 用户提供任何第三方服务密钥
- 用户修改关键配置参数
